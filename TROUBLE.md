## T-001: update-ranking.yml の YAML 構文エラー

- **発生日**: 2026-04-05（初回失敗を確認）
- **発見者**: nekorira-lab
- **対象ファイル**: `.github/workflows/update-ranking.yml`
- **症状**: GitHub Actions の Update Weekly Ranking ワークフローが毎回失敗し、ranking.js が自動更新されなかった
- **原因**: ワークフロー内の JavaScript テンプレートリテラルの継続行（`ranking.js —` などの行）のインデントが浅く、YAML パーサーがブロックの終了と誤認していた（103行目付近）
- **対処方法**: テンプレートリテラルを廃止し、`lines.push()` による行ごとの文字列構築に変更。日本語文字は Unicode エスケープ（`\uXXXX`）で記述してYAML上の問題を回避
- **対処日**: 2026-04-12
- **ステータス**: 解決済み

## T-002: ランキングページがブラウザ更新後も古いデータを表示

- **発生日**: 2026-04-12
- **発見者**: nekorira-lab
- **対象ファイル**: `docs/assets/js/ranking.js`（GitHub Pages キャッシュ）
- **症状**: ranking.js 更新後、ブラウザの更新ボタン（F5）を押しても古いデータ（3月23日〜29日）が表示され続けた
- **原因**: GitHub Pages は CDN キャッシュが効くため、通常の F5 ではキャッシュが残る
- **対処方法**: **Ctrl + F5**（強制リロード）でキャッシュを無視して最新ファイルを取得する
- **対処日**: 2026-04-12
- **ステータス**: 解決済み（仕様として運用で対応）

## T-003: 月次先行登録でpurchaseLinksが設定されずカードに予約ボタンが表示されない

- **発生日**: 2026-04-12
- **発見者**: nekorira-lab
- **対象ファイル**: `docs/assets/js/games.js`、`game-calendar-site/AGENTS.md`
- **症状**: 一部のゲームカード（例: 007 First Light）にAmazonの予約ボタンが表示されず、「Amazonで探す」「公式サイト」ボタンのみになった
- **原因**: 月次先行登録モード（AGENTS.md）の手順に `purchaseLinks` の調査・設定ステップが記載されておらず、エージェントがAmazon検索をスキップした。Amazon JPに予約ページが存在するにもかかわらず未設定のままになっていた
- **対処方法**:
  1. `007-first-light`（PS5/XBX/PC・5月27日発売）に `purchaseLinks` を追加（PS5通常版: B0G39D64ZH、PS5スペシャリストエディション: B0FZS6LBTB）
  2. `007-first-light-late-2026` を `007-first-light-sw2` に改め、発売日を `null` → `2026-06-18` に確定、Switch 2版 `purchaseLinks` を追加（B0FZRMWNL6）、「日程未定」セクションから6月セクションに移動
  3. `AGENTS.md` の月次モードセクションに「purchaseLinks の設定は月次モードでも必須」の注意書きを追記し再発防止
- **対処日**: 2026-04-12
- **ステータス**: 解決済み
