# ゲーム発売カレンダー — データ更新エージェント指示書

## あなたのミッション

`docs/assets/js/games.js` に最新のゲーム発売情報を追加してください。
毎週の発売タイトルを **ファミ通 発売スケジュール**（https://www.famitsu.com/schedule）から収集し、
`games.js` に追記します。

## 毎週の更新手順

### ステップ 1: 対象週を設定する

`game-calendar-site/tools/compare-week.js` の先頭にある変数を書き換える:

```js
const COMPARE_WEEK_START = 'YYYY-MM-DD';  // 対象週の月曜日
```

### ステップ 2: ファミ通からタイトルを収集する

https://www.famitsu.com/schedule を参照し、対象週（月〜日）の発売タイトルを確認する。

同ファイルの `REFERENCE_GAMES` 配列に転記する:

```js
const REFERENCE_GAMES = [
  { date: "YYYY-MM-DD", title: "タイトル名", platforms: ["SW2", "PS5"] },
  // ...
];
```

**title 表記ルール（重要）:**
- 日本語公式タイトルがある → 日本語で書く（例: `"ライフ イズ ストレンジ リユニオン"`）
- 英語のみ → 英語のまま（例: `"The Midnight Walk"`）
- 数字・記号は半角

**platforms の値:**
`SW2`（Switch 2）/ `SW1`（Switch）/ `PS5` / `PS4` / `XBX`（Xbox）/ `PC` / `MULTI`

**platforms の収集ルール（重要）:**
- ファミ通のページには機種ごとに列が分かれている。**同じタイトルが複数機種に掲載されている場合は、すべての機種を `platforms` 配列に含める**
- 例: SW2版とPS5版が同日発売 → `platforms: ["SW2", "PS5"]` として1エントリにまとめる
- 機種ごとに発売日が異なる場合のみ、別エントリに分ける
- **ファミ通に1機種しか掲載されていないタイトルは、必ず後述の「プラットフォーム3段階照合」を実施する**。他機種版が先行発売済みの場合はすべて `platforms` に追加する

### ステップ 3: 比較スクリプトを実行する

```sh
cd game-calendar-site
node tools/compare-week.js
```

出力を確認し、「❌ 抜けタイトル」に列挙されたエントリを `games.js` に追加する。

### ステップ 4: games.js にエントリを追追加する

出力末尾の「games.js 追記用コード」をコピーし、`games.js` の該当月コメント直前に貼り付ける。

**必ず埋めるフィールド:**

| フィールド | 説明 |
|---|---|
| `id` | 英数字ハイフン形式。スクリプトが自動生成するが日本語タイトルは `"game-YYMMDD"` になるので適宜修正 |
| `title` | タイトル名（表記ルール参照） |
| `date` | `"YYYY-MM-DD"` 形式。未定は `null` |
| `platforms` | 配列 |
| `genre` | 自由テキスト（例: `"アクション"`, `"RPG"`, `"アドベンチャー"`, `"シミュレーション"` など） |
| `note` | 海外先行発売・特記事項があれば記述、なければ `null` |
| `link` | 公式サイトURL。なければ `null` |
| `highlight` | 注目タイトルは `true`、通常は `false` |
| `purchaseLinks` | 機種ごとのAmazon Japan URL（後述） |
| `releasedPlatforms` | 先行発売済み機種のキー配列。該当なければ省略。例: `["PS5", "XBX"]`。ボタンが「購入」表示になる |
| `sourceUrl` | `"https://www.famitsu.com/schedule"` |
| `sourceName` | `"ファミ通"` |
| `lastVerifiedAt` | 今日の日付 `"YYYY-MM-DD"` |

### プラットフォーム照合ルール

#### 基本方針

| ステップ | 情報源 | 用途 |
|---|---|---|
| 一次 | ファミ通スケジュール | 発売日・プラットフォームの基準データ |
| 二次 | 4Gamer（https://www.4gamer.net/） | クロスチェック・差分確認 |
| 追加確認 | 各公式ストア | 下記トリガーに該当した場合のみ |

#### 通常フロー（大半のタイトル）

1. ファミ通で発売日・プラットフォームを取得する
2. 4Gamer でタイトルを検索し、プラットフォーム表記が一致するか確認する
3. 一致していれば **そのままデータに反映する**（公式ストア確認不要）
4. `sourceName` は `"ファミ通 / 4Gamer"` とする

#### 公式ストア確認トリガー（以下のいずれかに該当する場合のみ）

- ファミ通と4Gamerで **発売日が一致しない**
- ファミ通と4Gamerで **機種表記が一致しない**
- タイトルに「新ハード版・移植版・完全版・リマスター版」の可能性がある
- **先行発売済み機種が混ざっている疑い**がある（例: Switch 2版が新発売なのにPS5版も存在する）

トリガーに該当した場合は以下の公式ストアで確認する:

| 機種 | 確認先 |
|---|---|
| Switch / Switch 2 | https://store-jp.nintendo.com/ |
| PS5 / PS4 | https://store.playstation.com/ja-jp/ |
| Xbox | https://www.xbox.com/ja-JP/games |
| PC | https://store.steampowered.com/ |

#### 先行発売済み機種が混在する場合の反映ルール

公式ストア確認により「他機種版が先行発売済み」と判断した場合:

- 先行発売済み機種も `platforms` に追加する（ファミ通掲載機種を先頭にする）
  - 例: `platforms: ["SW2", "PS5", "XBX"]`
- `releasedPlatforms` に先行発売済み機種のキーを列挙する
  - 例: `releasedPlatforms: ["PS5", "XBX"]`
  - これによりボタンの表示が自動で「購入」に切り替わる
- `note` に先行発売の旨を記載する（機種名は書かない）
  - 例: `"後発対応機種あり（一部機種は先行発売済み）"`
- 先行発売済み機種の `purchaseLinks` も追加する

#### 解決できない場合

ファミ通・4Gamer・公式ストアを確認してもプラットフォームや発売日が特定できない場合は **手動補正対象** とし、`note` に `"要確認"` と記載してスキップする。

### purchaseLinks・エディション確認の手順

各タイトルについて **Amazon Japan（https://www.amazon.co.jp）のみ** を使って以下を確認する。
他のサイト（楽天・ヨドバシ・メーカー公式など）は参照しない。

#### ステップA: Amazon で検索する

`https://www.amazon.co.jp/s?k=タイトル名+機種名` で検索し、**出品者・出荷元がともに「Amazon.co.jp」** の商品のみを対象とする（マーケットプレイス出品者・他社倉庫からの発送は除く）。この条件は `purchaseLinks` に設定するURLにも同様に適用する。

#### ステップB: エディションの確認

検索結果に**複数のエディション**（例: 通常版・デラックスエディション・Ultimate Edition など）が存在する場合、以下のルールに従う。

**発売日が異なるエディションは別エントリにする:**
- 各エントリの `date` は Amazon 商品ページに記載された発売日を使用する
- `note` フィールドにエディション名を記載する（例: `"デラックスエディション"`）
- `id` にもエディションを反映する（例: `"pragmata-ps5-deluxe"`）
- 通常版（先行発売でない方）は `highlight: true`、デラックス等は `highlight: false`

**発売日が同じエディションは1エントリにまとめる:**
- 同日発売の通常版とデラックス版は別エントリにしない
- 通常版の `purchaseLinks` に標準版のURLを設定すれば十分
- デラックス版のASINを追加登録しない

**複数機種が同日発売の場合は1エントリにまとめる:**
- 例: PS5版とSW2版が同じ日付 → `platforms: ["PS5", "SW2"]` として1エントリ
- `purchaseLinks` に各機種のURLをまとめて設定する

エディションが1種類しかない場合は通常通り1エントリでよい。

#### ステップC: purchaseLinks を設定する

```js
purchaseLinks: {
  SW2:        "https://www.amazon.co.jp/dp/XXXXXXXXXX",  // 標準版
  PS5:        "https://www.amazon.co.jp/dp/XXXXXXXXXX",  // 標準版
  SW2_deluxe: "https://www.amazon.co.jp/dp/XXXXXXXXXX",  // デラックス版
  PS5_deluxe: "https://www.amazon.co.jp/dp/XXXXXXXXXX",  // デラックス版
},
```

デラックス版は `機種名_deluxe` キーで追加する。ボタンには自動で「（DX）」が付く。

**`_deluxe` キーを使う場面:**
- 標準版と同じエントリにデラックス版リンクを追加する場合 → `SW2_deluxe` を追加
- エントリ自体がデラックス版専用（発売日が標準版と異なる）の場合 → `PS5_deluxe` のみ設定し `PS5` キーは使わない（ボタンに「（DX）」が付き、標準版ボタンは表示されない）

- URLは `/dp/XXXXXXXXXX` 形式の商品ページURLを使用する（検索結果ページのURLは不可）
- Amazon に商品ページが見つからない場合はそのキーを省略する
- パッケージ版・ダウンロード版が両方ある場合はパッケージ版を優先する

**`genre` の目安:**

| ジャンル | 例 |
|---|---|
| アクション | マリオ、ゼルダ系 |
| RPG | ファイナルファンタジー、ドラクエ系 |
| アクションRPG | モンハン系 |
| アドベンチャー | ストーリー重視系 |
| シミュレーション | ストラテジー、育成系 |
| パズル | パズルゲーム全般 |
| レース | マリオカート系 |
| シューター | FPS/TPS系 |
| ビジュアルノベル | ノベルゲーム |
| 乙女ゲーム | 乙女向け恋愛ゲーム |
| スポーツ | サッカー、野球など |
| 格闘 | 格闘ゲーム |
| ホラーアドベンチャー | ホラー系 |

### ステップ 5: 再確認

```sh
node tools/compare-week.js
```

「抜けタイトル: 0 件」になれば OK。

## 注意事項

- `games.js` 以外のファイルは編集しない（`main.js`, `index.html`, `style.css` は触らない）
- `genre` フィールドを `""` のまま残さない
- `id` が既存エントリと重複しないよう確認する
- 全週の更新が完了したら最後にまとめてコミットする
- **プラットフォームの漏れに注意**: `id` に機種名（`-sw2` など）を含めない。1タイトル1エントリを原則とし、すべての対応機種を `platforms` に列挙する
- **`note` フィールドに機種名を書かない**（`"Switch 2版"` のような記述は不要）。機種情報は `platforms` で管理する
- **他機種版が先行発売済みのタイトル**: 「プラットフォーム照合ルール」の公式ストア確認トリガーに該当する場合のみ調査する。先行発売済み機種は `platforms`・`releasedPlatforms`・`purchaseLinks`・`note` にそれぞれ反映する（詳細は上記セクション参照）
